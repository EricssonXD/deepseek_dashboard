# Product

## Register

product

## Users

Individual developers and team managers using the DeepSeek API. They need to monitor spending across multiple API keys, break down costs by model and token type, and identify cost anomalies. Context: fast, often during development or monthly review. Primary job: answer "where is my money going?" in under 10 seconds.

## Product Purpose

A self-serve cost analytics dashboard for DeepSeek API usage. Upload exported billing data (ZIP) or fetch directly via API with a token. See total cost, per-key breakdowns, daily trends, model distribution, and token-type splits. Exists because the DeepSeek platform's native usage page lacks multi-key aggregation, historical comparison, and shareable views.

## Brand Personality

Professional, composed, minimal. Like Vercel Analytics — dark, typography-driven, data-first. No ornament. Trust through clarity, not decoration.

Three words: **sharp, restrained, legible.**

## Anti-references

- **Not gamified.** No badges, streaks, celebrations, confetti.
- **Not Notion.** No cover images, emoji-heavy, pastel, or whimsical.
- **Not AWS Console.** No dense tables, information overload, or nested navigation.
- **Not Datadog.** Not enterprise-busy or configuration-heavy.
- **No SaaS-cream.** Not warm-tinted backgrounds, no gradient buttons, no glass cards.
- **No AI-sloped dashboard clichés.** No hero-metric template with gradient accent, no identical icon+heading+text card grids as decoration, no tiny uppercase tracked eyebrows on every section.

## Design Principles

1. **Data first, ornament never.** Every pixel serves comprehension. Remove anything that doesn't help answer a cost question.
2. **Typography-driven hierarchy.** Clarity through weight, size, and spacing — not through colored badges, icons, or decoration.
3. **Fast scan, fast act.** Dashboard users don't read; they scan. Layout must surface the answer before the question is fully formed.
4. **Restrained dark.** Dark theme for long-screen sessions. Not black-void harsh; layered grays with enough contrast to breathe.
5. **One screen, one story.** Each view answers exactly one cost question. No scrolling to piece together a narrative.

## Accessibility & Inclusion

Standard basic usability. No specific WCAG target set, but maintain readable contrast and keyboard-navigable controls as baseline.
